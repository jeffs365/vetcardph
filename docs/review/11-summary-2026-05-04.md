## Highest-Priority Items Now

These are the items that should be tackled first based on impact × effort:

### Critical — security & correctness
1. **API N1** — `GET /pets/:petId/preventive-records` is missing `clinicId` filter. Cross-clinic data leak in the shared-pet flow. (`backend/src/routes/preventive-records.ts:168`)
2. **Web N9** — JWT in `localStorage` exposes auth to any future XSS. Plan httpOnly cookie migration. (`web/src/lib/auth.tsx:35`)
3. **API N5** — Staff login has no brute-force protection (only owner-side does). (`backend/src/routes/auth.ts:112`)

### High — production readiness
4. **DB Issue 8** — No Prisma migrations directory. Switch to `prisma migrate dev` and commit a baseline before multi-instance prod.
5. **API Prev 5 / N4** — Owner data overwrite is only blocked for *claimed* owners. Most owners are unclaimed. Tighten guard to require existing clinic linkage.
6. **API N3** — Appointment status transitions are unvalidated; `COMPLETED → SCHEDULED` orphans the linked Visit.
7. **API N2** — No role check on appointment endpoints — any authenticated staff member can mutate. Decide policy and enforce.

### High — daily UX
8. **Web N1, N3** — Mobile input attributes (`inputMode`, `autoComplete`, `capture="environment"`). Tiny diff, big quality-of-life on iOS/Android.
9. **Web N14** — Global API request timeout. Prevents indefinite hangs on slow clinic WiFi.
10. **Web N15** — Color-only status indicators fail color-blind users (also a WCAG concern).

### Medium — scaling
11. **API Prev 2** — Pet age filter in JS after fetching 250 rows. Add an `ageMonths` column and filter in SQL.
12. **API Prev 8** — Cursor pagination on `GET /pets`.
13. **DB N3, N4, N5** — Composite indexes for actual query patterns: `PreventiveRecord [clinicId, petId, careTypeId, administeredOn]`, `Visit [clinicId, petId, visitDate]`, `Staff [clinicId, isActive]`.
14. **DB N1** — Cleanup job for expired `OwnerOtp` and `ShareToken` rows.
